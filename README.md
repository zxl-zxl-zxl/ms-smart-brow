# MS 智能画眉

MS 智能画眉是一款基于微信小程序的画眉辅助工具。当前版本以「拍照定标 + 实时辅助 + 自动定标兜底」为核心流程：用户打开前置相机后，手动触发一次人脸定标，系统根据端侧人脸关键点生成眉形辅助线和推荐眉形；画眉过程中可默认开启实时辅助，让辅助层随脸部小幅平移、缩放和旋转，脸部位置或大小变化超过阈值时自动用当前人脸结果重新定标。用户可切换模板、微调辅助层，并保存不带辅助线的净图到相册。

## 项目状态

当前代码更接近 V1/MVP 实现：

- 已实现首页、相机定标页、保存成功页、个人中心、隐私说明页、关于页。
- 已接入微信小程序端侧 `faceDetect` 能力，用于单帧人脸检测和关键点定位。
- 已实现定标后的实时辅助开关，支持低频人脸检测、辅助层跟随和阈值触发自动定标。
- 已实现 4 种眉形模板：自然眉、标准眉、平眉、弯月眉。
- 已实现基于脸部比例的规则推荐，不依赖大模型，也不上传照片做云端 AI 分析。
- 已开启微信基础分享能力，支持右上角转发给好友/群，首页支持分享到朋友圈。
- 已接入微信云开发用户初始化云函数 `initUser`，用于静默创建/同步用户身份。
- 会员、付费模板、历史记录、用户画像等能力目前仍属于规划或预留方向。

## 技术栈

- 跨端框架：Taro 4.2.0
- 前端框架：React 18
- 语言：TypeScript 5
- 样式：SCSS/Sass
- 构建器：Webpack 5（Taro 内置）
- 目标端：微信小程序
- 云能力：微信云开发、`wx-server-sdk`
- 核心端侧能力：`Camera`、`onCameraFrame`、`Taro.faceDetect`、`Taro.cloud.callFunction`

## 核心页面

### 首页 `src/pages/home`

首页负责产品入口和静默身份初始化：

- 点击「开始智能画眉」时先调用 `initUser()`。
- 用户初始化失败时提示「身份初始化失败，已进入体验模式」，仍进入相机页，保证核心功能可体验。
- 点击「我的」时同样先初始化用户身份；失败则停留首页。
- 页面展示当前可切换的眉形模板和隐私提示。
- 支持基础分享落地页，转发标题为「MS 智能画眉 - AI 相机画眉助手」。

### 相机页 `src/pages/camera`

相机页是核心交互页面，包含以下状态和能力：

- 启动前置摄像头，并通过 `createCameraContext().onCameraFrame` 缓存最新相机帧。
- 初始化微信端侧人脸检测能力 `Taro.initFaceDetect`。
- 点击「开始定标」后，使用当前单帧调用 `Taro.faceDetect`。
- 定标成功后生成辅助层，并默认开启「实时」辅助开关。
- 实时辅助开启时，后台低频检测最新相机帧；小幅变化时只平移、缩放、旋转当前辅助层，大幅且持续变化时自动定标，使用当前人脸结果重新生成辅助层。
- 实时辅助关闭时，辅助层保持固定，仅支持用户手动微调或重新定标。
- 展示脸部中线、双眼水平线、眉区参考框、关键点和眉形贴图。
- 支持打开/关闭辅助线、打开/关闭实时辅助、切换眉形模板、重新定标。
- 支持通过滑杆调整辅助层的上下、左右、大小、旋转和透明度。
- 点击「完成保存」会重新拍摄一张净图并保存到系统相册，然后进入保存成功页。

### 个人中心 `src/pages/profile`

个人中心负责展示云端用户身份状态：

- 进入页面后调用 `initUser()`。
- 展示会员状态、用户编号后 6 位。
- 提供隐私说明、关于页面入口。
- 会员功能当前仅为规划占位。

### 基础转发

所有页面接入统一的微信基础转发配置：

- 转发标题：MS 智能画眉 - AI 相机画眉助手。
- 转发描述：实时相机 + 人脸定标 + 眉形辅助线，对齐辅助线把眉头、眉峰、眉尾画准。
- 转发给好友/群时统一落地到首页。
- 首页支持分享到朋友圈；其他页面暂不开放分享到朋友圈，避免从非首页分享时落到当前页面。
- 转发内容不携带用户照片、人脸数据或画眉结果图。

## 核心模块

### 云调用 `src/modules/cloud`

封装微信云开发初始化和云函数调用：

- `initCloud()`：根据 `src/config/cloud.ts` 中的云环境 ID 初始化 `Taro.cloud`。
- `callCloudFunction<T>()`：统一调用云函数并返回 `result.result`。

### 用户初始化 `src/modules/user`

当前只封装了 `initUser()`，调用云函数 `initUser` 并返回用户资料。

云函数位于 `cloudfunctions/initUser`，主要逻辑：

- 通过 `cloud.getWXContext()` 获取微信 `OPENID`。
- 自动确保 `users` 集合存在。
- 如果用户已存在，则更新 `lastLoginAt` 和 `updatedAt`。
- 如果用户不存在，则创建 `userId`、会员状态和时间字段。
- 返回给小程序端时会移除 `openid` 字段。

### 人脸分析 `src/modules/faceAnalyzer`

负责把微信 `faceDetect` 的原始结果标准化为业务可用的 `FaceAnalysisResult`：

- 兼容 `faceInfo` 为数组、对象或根对象的返回结构。
- 标准化人脸中心、人脸框、关键点、角度和置信度。
- 做基础质量判断，包括无人脸、多张脸、距离过近/过远、脸部偏离中心、歪头过大。
- 计算脸型比例、人脸宽度占比、中心偏移、标准化人脸框等指标。
- 相机页会复用该模块支持实时辅助和自动定标；检测仍在端侧完成。

### 眉形引擎 `src/modules/browEngine`

负责把人脸关键点和模板参数转成屏幕上的辅助层数据：

- 将相机帧坐标映射到小程序视口坐标。
- 尝试使用固定关键点索引定位左右眉区域。
- 如果关键点不足，会根据脸框和比例回退生成眉形位置。
- 根据模板参数控制眉峰比例、弧度、粗细、角度和眉尾抬升。
- 对左右眉做镜像和尺寸校正，输出 `OverlayData`。

### 眉形推荐 `src/modules/recommendation`

当前使用规则算法推荐眉形：

- 脸部比例偏修长：推荐平眉。
- 脸部比例偏圆润：推荐标准眉。
- 脸部线条较舒展：推荐弯月眉。
- 比例较均衡：推荐自然眉。

## 数据与隐私边界

当前实现遵循 V1 隐私边界：

- 人脸检测在微信小程序端侧完成。
- 小程序不上传用户照片。
- 小程序不保存人脸关键点数据。
- 实时辅助和自动定标只使用当前会话内的端侧检测结果，不落库存储。
- 保存相册时重新拍摄并保存不带辅助线的净图。
- 云端仅保存用户身份相关信息，如 `userId`、`openid`、昵称、头像、会员状态和登录时间。

## 本地开发

### 环境要求

- Node.js >= 20
- npm
- 微信开发者工具
- 已开通并配置微信云开发环境

### 安装依赖

```bash
npm install
```

### 类型检查

```bash
npm run typecheck
```

### 构建微信小程序

```bash
npm run build:weapp
```

### 开发监听

```bash
npm run dev:weapp
```

构建产物输出到 `dist/`。在微信开发者工具中导入项目时，通常选择本项目根目录，并根据 Taro/微信开发者工具配置预览 `dist` 产物。

## 云开发配置

云环境 ID 配置在：

```ts
// src/config/cloud.ts
export const cloudConfig = {
  env: 'cloud1-d4gmfknrz08ee27da',
}
```

如需切换环境，修改 `cloudConfig.env` 后重新构建。云函数代码位于 `cloudfunctions/initUser`，部署到同一云环境后，小程序端才能正常调用个人中心和静默用户初始化能力。
